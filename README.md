# Song2Radio 音乐播报

把网易云音乐和 QQ 音乐的网页歌单，变成一档带口播串场的私人电台。

Song2Radio 是一个 Chrome / Edge 浏览器插件。它会在每首歌播放前暂停音乐，读取当前歌曲信息，生成一段简短中文介绍，再用系统语音或云端音色播报出来。播报结束后，歌曲继续播放。

如果你喜欢认真听一张专辑、一份歌单，或者希望每首歌之间多一点“电台主持人”的仪式感，这个插件就是为这个场景做的。

## 功能特点

- 支持网易云音乐网页版：`music.163.com`
- 支持 QQ 音乐网页版：`y.qq.com`
- 每首歌播放前自动播报歌名、歌手、专辑等信息
- 支持第一首歌开场白和歌单最后一首歌收尾语
- 可使用浏览器 / 系统自带语音，免费、低延迟
- 可接入阿里云百炼，生成 AI 电台文案和云端 TTS 音色
- 可选择 DeepSeek 作为文案生成模型
- 接口失败时自动降级为简单文案，不影响正常听歌
- 云端口播支持淡入淡出，让歌曲进入更自然

## 当前状态

这是一个自用项目整理出来的早期版本，目前还没有上架 Chrome 应用商店，需要手动加载插件。

请注意：

- 插件不内置任何作者 API Key。
- 如果不填写 API Key，仍然可以使用简单文案 + 系统语音。
- 如果需要 AI 文案或百炼云端音色，需要填写你自己的 API Key。
- AI 生成的歌曲背景可能出错，尤其是冷门歌曲，请把它当作“辅助介绍”，不要当作资料库。

## 安装教程

下面以 Chrome 为例，Edge 的操作基本一样。

### 第一步：解压插件

如果你下载的是 `netease-intro-extension.zip`，请先解压。

解压后你会得到一个文件夹：

```text
netease-intro-extension/
```

请确认这个文件夹里能看到这些文件：

```text
manifest.json
background.js
popup.html
popup.js
bridge.js
intercept.js
adapters/
```

### 第二步：打开扩展程序页面

在浏览器地址栏输入：

```text
chrome://extensions/
```

如果你使用 Edge，也可以输入：

```text
edge://extensions/
```

### 第三步：打开开发者模式

在扩展程序页面右上角，打开「开发者模式」。

### 第四步：加载插件

点击「加载已解压的扩展程序」，选择刚才解压出来的：

```text
netease-intro-extension/
```

加载成功后，浏览器右上角会出现「音乐播报」插件。

建议把它固定到工具栏，后面开关和配置会方便很多。

## 快速开始

### 网易云音乐

1. 打开网易云音乐网页版：`https://music.163.com/`
2. 播放任意歌单或专辑。
3. 插件会在歌曲播放前暂停音乐。
4. 播放一段口播介绍。
5. 口播结束后，歌曲继续播放。

### QQ 音乐

1. 打开 QQ 音乐网页版：`https://y.qq.com/`
2. 播放任意歌单或专辑。
3. 插件会等到 QQ 音乐真正开始播放后，先暂停歌曲。
4. 播放第一首歌的开场白和歌曲介绍。
5. 口播结束后，歌曲继续播放。

如果你第一次打开页面时没有听到播报，可以刷新音乐页面，或在扩展程序页面点击插件的「重新加载」。

## 插件设置说明

点击浏览器右上角的「音乐播报」图标，可以看到几个设置项。

### 播放前语音播报

这是总开关。

打开后：每首歌播放前都会尝试播报介绍。

关闭后：插件不再打断播放，音乐网站恢复原本行为。

### AI 生成介绍

打开后：插件会尝试调用你配置的模型，生成更像电台主持人的口播文案。

关闭后：插件只使用简单模板，例如：

```text
接下来为您播放：歌曲名，演唱：歌手名
```

### 音色

你可以选择三类声音。

```text
浏览器默认（兜底）
```

使用浏览器自带的语音朗读。它不需要 API Key，适合先体验和长期免费使用。

```text
系统语音（免费）
```

插件会列出系统里可用的语音。不同电脑可选声音不同。

```text
百炼云端音色
```

使用阿里云百炼 TTS 音色，声音更像播客或电台，但需要填写百炼 API Key。

### 百炼 API Key

百炼 Key 有两个用途：

- 使用通义千问生成 AI 口播文案。
- 使用百炼云端音色生成语音。

不填也可以使用插件，只是会退回简单文案和系统语音。

### 用 DeepSeek 替代生成文案

如果打开这个开关，文案生成会改用 DeepSeek。

注意：DeepSeek 只负责生成文字。如果你选择的是百炼云端音色，语音合成仍然需要百炼 API Key。

## API Key 配置

### 不配置 Key 的用法

如果你只是想先试试插件，可以这样设置：

1. 打开「播放前语音播报」。
2. 关闭或打开「AI 生成介绍」都可以。
3. 音色选择「浏览器默认」或「系统语音」。
4. 百炼 API Key 和 DeepSeek API Key 都留空。

这种模式下，插件会使用简单文案播报，不产生 API 费用。

### 使用百炼

如果你想使用通义千问文案和百炼云端音色：

1. 准备自己的阿里云百炼 API Key。
2. 点击插件图标。
3. 在「百炼 API Key」输入框中填入 Key。
4. 点击「保存」。
5. 打开「AI 生成介绍」。
6. 音色选择一个「百炼云端」音色。

### 使用 DeepSeek 写文案

如果你想用 DeepSeek 生成口播文案：

1. 打开「用 DeepSeek 替代生成文案」。
2. 填入自己的 DeepSeek API Key。
3. 点击「保存」。
4. 打开「AI 生成介绍」。

如果此时音色选择系统语音，那么只需要 DeepSeek Key。

如果此时音色选择百炼云端音色，那么还需要百炼 Key 来合成语音。

## 推荐设置

### 免费稳定模式

适合日常长期使用。

```text
播放前语音播报：开启
AI 生成介绍：关闭
音色：系统语音或浏览器默认
API Key：不填
```

优点是免费、快、稳定。

缺点是文案比较简单。

### AI 文案 + 系统语音

适合想听更自然介绍，但不想使用云端音色的人。

```text
播放前语音播报：开启
AI 生成介绍：开启
音色：系统语音或浏览器默认
百炼 API Key 或 DeepSeek API Key：填写一个
```

优点是成本相对低，延迟也比较可控。

### 完整电台模式

适合想获得更像电台节目效果的人。

```text
播放前语音播报：开启
AI 生成介绍：开启
音色：百炼云端音色
百炼 API Key：填写
```

优点是文案和声音都更完整。

缺点是会调用云端接口，可能产生费用，也会有一点等待时间。

## 常见问题

### 为什么没有播报？

请按顺序检查：

1. 插件是否已经加载成功。
2. 音乐网站页面是否刷新过。
3. 插件里的「播放前语音播报」是否开启。
4. 浏览器是否允许网页播放声音。
5. 是否正在使用网易云音乐网页版或 QQ 音乐网页版。

### 为什么只有简单的“接下来为您播放”？

通常是因为没有配置 API Key，或者 AI 接口请求失败。

如果你希望生成更完整的电台文案，请填写百炼 API Key 或 DeepSeek API Key，并打开「AI 生成介绍」。

### 为什么云端音色没有声音？

请检查：

1. 是否填写了百炼 API Key。
2. 选择的是否是百炼云端音色。
3. API Key 是否仍然有效。
4. 百炼账户是否有可用额度。

如果云端音色失败，插件会自动尝试退回系统语音。

### 为什么 QQ 音乐第一首歌以前会直接播放？

QQ 音乐页面会先显示第一首歌信息，再真正进入播放状态。旧版本可能会在“歌名刚显示出来”时提前触发播报，导致用户真正点击播放时，第一首歌已经被插件认为处理过。

现在的逻辑改成：必须等 QQ 音乐真实处于播放状态后，插件才触发播报；触发后先暂停歌曲，再播放开场白，最后恢复播放。

### 会不会泄露我的 API Key？

插件会把你填写的 Key 保存在浏览器本地扩展存储中，用于请求你选择的模型和 TTS 服务。

请注意，浏览器插件前端环境不适合内置公共 Key，所以本项目不会内置作者 Key。你也不要把自己的 Key 写死进源码后公开发布。

## 实现原理

项目采用 Manifest V3 浏览器插件结构。

```text
netease-intro-extension/
  manifest.json
  background.js
  popup.html
  popup.js
  bridge.js
  intercept.js
  adapters/
    netease.js
    qq.js
```

### 核心流程

1. 页面适配器读取当前歌曲信息。
2. 拦截器在歌曲播放前暂停音乐。
3. 页面脚本通过 `window.postMessage` 把歌曲信息发给桥接脚本。
4. 桥接脚本调用扩展后台 `background.js`。
5. 后台根据配置生成文案，必要时生成云端语音。
6. 页面播放口播。
7. 口播结束后恢复歌曲播放。

### 网易云音乐适配

网易云音乐主要通过拦截页面里的 `HTMLMediaElement.prototype.play` 来判断新歌播放。

读取歌曲信息时，优先使用页面播放器对象；如果失败，再尝试从播放栏 DOM 和页面标题里读取。

### QQ 音乐适配

QQ 音乐使用页面观察模式。

插件会轮询播放器底部区域，观察当前歌曲是否发生变化。为了避免第一首歌提前触发，插件只会在 QQ 音乐已经处于播放状态时开始处理。

### 文案生成

后台会把歌名、歌手、专辑等信息组织成提示词，让模型生成一段 120 字以内的中文电台口播。

如果是第一首歌，会要求模型加自然开场白。

如果是最后一首歌，会要求模型加温暖收尾语。

如果模型失败或超时，会退回简单模板。

### 语音播放

系统语音使用浏览器的 `speechSynthesis`。

百炼云端音色使用百炼 TTS 接口，返回音频地址后在页面播放。

网易云音乐中，云端音频还会在结尾阶段和歌曲做简单淡入淡出，让过渡更自然。

## 开发说明

这个项目没有构建步骤，源码就是插件本体。

修改代码后，重新打包：

```powershell
Compress-Archive -Path .\netease-intro-extension\* -DestinationPath .\netease-intro-extension.zip -Force
```

检查 JavaScript 语法：

```bash
node --check netease-intro-extension/background.js
node --check netease-intro-extension/intercept.js
node --check netease-intro-extension/adapters/netease.js
node --check netease-intro-extension/adapters/qq.js
node --check netease-intro-extension/bridge.js
node --check netease-intro-extension/popup.js
```

检查是否误提交 API Key：

```bash
rg -n "sk-[A-Za-z0-9]|DEFAULT_API_KEY|freeCallsUsed|quotaAnnounced|FREE_CALL_LIMIT" .
```
